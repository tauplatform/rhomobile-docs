<p>The MediaPlayer extension is used to playback different types of audio files. It also allows the playback of Ringtones for the respective platform.</p>
<h2>Enabling the API</h2>

<p>In order to use this API you must include the following extension in your <code>build.yml</code>
    :::ruby
    extensions: [&ldquo;mediaplayer&rdquo;]</p>

<h2><i class='icon-warning-sign'></i>Remarks</h2>

<a name='r0'></a><div class=' remarks' id='r0'><div class="accordion-group"><div class="accordion-heading"><span class="accordion-toggle"  href="#cRemark0"><strong>Availbility of Operating System</strong></div><div id="cRemark0" class="accordion-body">  <div class="accordion-inner"><p>Because the MediaPlayer extension uses the operating system&rsquo;s built-in player for some media types, it may be possible for the user to access operating system features while media is playing.</p>
  </div></div></div></div><a name='r1'></a><div class=' remarks' id='r1'><div class="accordion-group"><div class="accordion-heading"><span class="accordion-toggle"  href="#cRemark1"><strong>Playback Controls</strong></div><div id="cRemark1" class="accordion-body">  <div class="accordion-inner"><p>Because the MediaPlayer extension uses the operating system&rsquo;s built-in player for some media types, the playback controls available to the user are outside the control of RhoElements. Some players may have no playback controls in which the user will have to wait for the media to finish playing.</p>
  </div></div></div></div><a name='r2'></a><div class=' remarks' id='r2'><div class="accordion-group"><div class="accordion-heading"><span class="accordion-toggle"  href="#cRemark2"><strong>Android Support</strong></div><div id="cRemark2" class="accordion-body">  <div class="accordion-inner"><p>On Android devices, HTTPS is supported only on Android versions 4.0 (Ice Cream Sandwich) or newer.</p>
  </div></div></div></div><a name='r3'></a><div class=' remarks' id='r3'><div class="accordion-group"><div class="accordion-heading"><span class="accordion-toggle"  href="#cRemark3"><strong>Using File Transfer</strong></div><div id="cRemark3" class="accordion-body">  <div class="accordion-inner"><p>When more flexible file transfer is required than is provided by MediaPlayer itself it is recommended to use the Network API to fetch the media file, and then to play the local file using MediaPlayer. For example, if playing a file from an FTP server, or an HTTP server requiring authentication. This can also be more efficient if the same file is to be played multiple times, especially on Windows Mobile/CE, where there is no streaming support and therefore the file is completely downloaded before being played. On Windows Mobile/CE devices, HTTPS file transfer is not supported.</p>
